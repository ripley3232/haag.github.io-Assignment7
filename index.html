BRUTALIST WEB DESIGN

Last Updated: November 10, 2025

DEFINITION

Brutalism in web design is the intentional attempt to create something raw, honest, and unpolished. It rejects the notion that websites must be beautiful, smooth, or user-friendly in the conventional sense. Instead, brutalist websites expose their structure, embrace visual harshness, and prioritize function over aesthetics.

"A website's materials aren't stone and concrete, but HTML, CSS, and JavaScript. Brutalism means using these materials honestly."

CORE PRINCIPLES

1. Raw HTML

Content is king. HTML should be semantic and minimal. Tables for layout. Lists as lists. No div soup. No framework bloat.

2. Honest Typography

System fonts only. Times New Roman. Arial. Courier. No custom web fonts. No subtle kerning. No optical illusions.

3. Visible Structure

Show the bones. Borders on everything. Tables with visible cells. Horizontal rules as design elements. The <hr> tag is your friend.

4. Default Colors

Black text on white background. Blue links (#0000EE). Purple visited links (#551A8B). Maybe some red for emphasis. That's it.

5. No Decoration

No gradients. No shadows. No rounded corners. No animations. No parallax scrolling. No hero images. No carousels. Nothing.

WHY BRUTALISM?

  • Reaction to homogeneity - Everything looks like Bootstrap or Material Design
  • Speed - Raw HTML loads instantly. No 2MB of JavaScript frameworks.
  • Accessibility - Semantic HTML works with screen readers
  • Honesty - The web was built for sharing documents, not selling products
  • Rebellion - Against corporate design systems and visual conformity
  • Statement - Ugliness as art. Function over form.

EXAMPLES IN THE WILD

Real websites that embrace brutalist principles:

  1. Irish Wrecks Online - Pure database functionality
  2. Yale School of Art (archived) - Chaotic, user-editable, intentionally broken
  3. Craigslist - Accidental brutalism through pure utility
  4. Berkshire Hathaway - Warren Buffett's deliberately minimal site

HOW TO CREATE BRUTALIST DESIGN

Step Action
1 Write semantic HTML with no divs unless necessary
2 Use tables for layout (yes, really)
3 Apply minimal CSS - system fonts, basic colors
4 Show borders on structural elements
5 Remove anything "pretty" or "smooth"
6 Test by asking: "Is this ugly enough?"

CRITICISM

Brutalist web design has its detractors:

  • Poor usability for general audiences
  • Inaccessible to people with certain disabilities
  • Self-indulgent and pretentious
  • Confuses "bad design" with "honest design"
  • Works for art projects, fails for real businesses

These criticisms are valid. Brutalism is not appropriate for every website. It's a design philosophy, not a universal solution.

CODE EXAMPLE

<!DOCTYPE html>
<html>
<head>
    <title>Simple Page</title>
</head>
<body>
    <h1>Header</h1>
    <p>Content</p>
    <a href="#">Link</a>
</body>
</html>

That's it. No CSS. No JavaScript. Just HTML. Pure brutalism.

CONCLUSION

This webpage demonstrates brutalist web design by using the principles it describes. Notice the Times New Roman font. The black borders on tables. The stark color scheme. The horizontal rules as section dividers. The default link colors. The complete absence of visual polish.

Is it ugly? Yes. Is it honest? Also yes.

Brutalism isn't about making bad websites. It's about questioning what makes a website "good" in the first place. It's about remembering that the web was built for sharing information, not for selling things or looking impressive.

The web doesn't need to be pretty to be useful.
